# DESIGN.md - casoCAD Design Reference

This document defines what casoCAD is, the exclusive set of libraries it may
use, how the workspace is organized, the current status, and the future view.
The root `README.md` stays a short quick-start; this file is the deep
reference.

## 1. Project Objective

casoCAD is an SDF-based CAD application for authoring solver-ready 2D and 3D
CFD/FEA analysis cases, written in Rust and targeting **native desktop and
the web browser from a single codebase** via wgpu.

This Rust workspace began as "casoWASM", a full port of the original Python
casoCAD application (PySide6 + QRhi). The port is complete and the Python
codebase has been removed; its final snapshot is archived at the
`python-final` tag (branch `final-python`, commit `6185ec6`).

The parity rule that framed the port still stands:

- **Parity is demonstrated, not assumed.** Golden files generated by the
  Python kernel are frozen in-repo and replayed by Rust tests; scenes saved
  by this app round-trip through the archived Python app unchanged.

What ports is the **current** Python architecture: the exact-SDF kernel and
the CPU-tessellated display-surface viewport (analytic primitive tessellation,
exact boolean clipping, dual contouring) rendered with a handful of static
shaders. The deleted GPU raymarcher / shader codegen is *not* resurrected.

## 2. Binding Principles

- `#![forbid(unsafe_code)]` in every crate, enforced through
  `[workspace.lints]` in the root `Cargo.toml`.
- **f64 for all kernel/analysis math; f32 only at the GPU upload boundary.**
- Terminology carries over from the Python codebase:
  - boolean operations are **SDF operators** — never "CSG";
  - **"meshing" is reserved for FEA/CFD** (the `meshing/` crate, the Meshing
    workspace, `MeshableDomain`). The viewport makes **surfaces** (surface
    builders, tessellation, display surfaces) and must never be called a
    mesher.
- The working unit (km/m/cm/mm) is display-only: the model is always meters;
  a unit switch rescales the camera/grid, never committed geometry.
- The exactness role system ("safe geometry compiler", spec
  `design_docs/exact_signed_distance_field_cfd_migration_v2.md`) is ported
  behaviorally: the structural validation walk matches Python exactly.

## 3. Exclusive Library Set

Minimal dependencies are a design constraint, not a preference. A new
dependency is admitted only if it is irreplaceable for a target (GPU, UI,
browser interop, Arrow interop). Everything else is hand-rolled.

Per crate, the complete allowed set:

| Crate | External dependencies |
| --- | --- |
| `kernel` | `serde_json` (scene.json only) — nothing else |
| `surfaces` | **zero** |
| `render` | `wgpu 29`, `bytemuck` |
| `meshing` | `arrow-array` / `arrow-schema` / `arrow-ipc` / `arrow-buffer` (v56), `serde_json` |
| `app` | `eframe 0.35` (with the `wgpu` feature), `egui_dock 0.20`, `egui_extras 0.35` (`svg` only, for the vector wordmark), `rhai 1`, `serde_json` |
| `app` (wasm32 only) | `wasm-bindgen`, `wasm-bindgen-futures`, `web-sys`, `js-sys`, `getrandom` (`wasm_js` backend), `log` |

Internal workspace dependencies flow strictly downward and are not restricted
by the table above: `meshing` uses `surfaces` for exact 2D outline arcs
(`boundary_outline`), on top of its `kernel` dependency.

Constraints and deliberate rejections:

- **`wgpu` in `render` must match the `wgpu` version bundled by `eframe`**
  (currently both 29). Bump them together, never independently.
- **No math crates** (`glam`, `nalgebra`): vectors, matrices, frames, and the
  3×3 symmetric eigensolver for the QEF are hand-written in `kernel` and
  `surfaces`. This keeps the kernel dependency-free and the formulas
  literally comparable with the Python source.
- **No `rayon` / threading in the shared crates**: builds must run on wasm32.
- Scripting is **Rhai**, not embedded Python — the Python-subprocess model of
  the desktop Meshing workspace cannot run in WASM.

## 4. Workspace Architecture

Dependency direction is strictly downward; `kernel` and `surfaces` never
touch GPU or UI.

```
<repo root>/
  kernel/      caso-kernel   — exact-SDF geometry kernel (no GPU/UI deps)
  surfaces/    caso-surfaces — CPU display-surface builders
  render/      caso-render   — wgpu renderer (WGSL shaders, camera)
  meshing/     caso-meshing  — FEA/CFD mesh artifacts (Arrow IPC export)
  app/         caso-app      — egui application; native bin + wasm cdylib
  web/         index.html + wasm-pack output (pkg/)
```

### 4.1 `kernel` — the exact-SDF geometry kernel

Port of `core/`. Modules:

- `sdf/` — every node kind: 3D primitives (`primitives_3d.rs`), 2D profiles
  and 1D constructions with their placement frames (`primitives_2d.rs`,
  `primitives_1d.rs`, `placed.rs`), tubes (`tubes.rs`), extrude/revolve
  (`solid_from_2d.rs`), `NormalCurtain` (`curtain.rs`), operators and
  transforms (`node.rs`). Batched f64 evaluation is the `to_numpy()` analog.
- `roles.rs` / `preconditions.rs` / `model.rs` — the exactness role system,
  generator/offset preconditions, and `compile_model` (grammar + preconditions
  + sampled Domain disjointness).
- `scene.rs` — the id-based scene document (ObjectId handles, payloads
  reference children by id, `build_node` produces the kernel tree; all edit
  ops: add/combine/wrap-transform/solid-from-2d/move/rotate/delete/rename/
  domains/duplicate/import-subtree; snapshot = Clone for undo).
- `boundary.rs` / `boundary_ops.rs` / `boundary_paths.rs` — Boundary Region
  v2: the classifier (owner-active + patch scope + cut-chain sign
  conjunction; `boundary_region_base_mask` is the criteria-1-3 half,
  `boundary_region_mask` composes it with the cut chain), analytic surface
  patches, sphere-trace picking with owner attribution, knife construction
  (segment = planar slice oriented by the mean endpoint normal;
  polygon/bezier point stencils sit on the mean-click-normal plane and
  extrude one-sidedly so closed surfaces never grow an antipodal phantom
  cut; a smooth on-surface polyline knife was removed 2026-07-12 as unproven
  — see `design_docs/boundary_cutter_exactness.md` for the archived record).
- `serialization.rs` — scene.json v1 read/write, including boundary-region
  records and ghost knives (leaf shapes plus the recursive `extrude` composite
  record for one-sided stencils).
- `meshing.rs` — the `MeshableDomain` / `MeshableBoundaryRegion` API loaded
  from a saved scene (the FEA/CFD-facing view of the document), plus the
  mesher toolkit queries: total boundary classification with owner
  attribution, and `MeshableInterface` (the exact shared wall between
  nested marked domains; see `design_docs/meshing_toolkit.md`).
- `differential.rs` — batch differential queries on exact fields: normals,
  interior-seeded Newton projection onto the boundary (positive starts are
  refused — the interior-exactness contract), curvature stencils.

### 4.2 `surfaces` — CPU display-surface builders

Port of the viewport surface pipeline (`surface_builder.py`,
`surface_clipping.py`, `surface_contouring.py`): analytic tessellation for
every leaf kind (`builders.rs`, with dispatch and a cache with translation
reuse), exact boolean clipping with root-found seams (`clipping.rs`), dense +
narrow-band dual contouring with a QEF solved by a hand-written Jacobi 3×3
eigensolver (`contouring.rs`), 2D fills via marching squares + ear clipping
(`profiles2d.rs`), and shared geometric ops (`geomops.rs`). Headless by
design — this crate never touches wgpu.

### 4.3 `render` — wgpu renderer

Orbit camera with `screen_ray` / `project_to_screen` as exact CPU/GPU
inverses (`camera.rs`); offscreen renderer (Rgba8Unorm + Depth32Float) with
grid → surface → line → point passes (`renderer.rs`); four WGSL shaders
(`surface.wgsl` per-vertex Lambert, `grid_axes.wgsl` fullscreen analytic
grid/axes, `line.wgsl` screen-space thick lines, `point_marker.wgsl`
instanced sphere impostors at constant pixel size).

Design invariant: the geometry opacity slider affects display surfaces
only — the Meshing-workspace preview overlay never fades with it, so
opacity ~0 doubles as an x-ray mesh-inspection mode. See
`design_docs/mesh_preview_opacity_independence.md`.

### 4.4 `meshing` — FEA/CFD mesh artifacts

Arrow IPC MeshIR v1 writer/reader. MeshIR is a single entity table with
shared `point`, `edge`, `face`, `cell`, `zone`, `tag`, and `attribute`
rows; cells reference shared topology instead of duplicating coordinates.
The schema supports point/edge/triangle/quad/polygon/tet/hex/prism/pyramid/
polyhedron element families, with world-space f64 xyz coordinates in meters.

The `toolkit` module is the mesher-facing base layer
(`design_docs/meshing_toolkit.md`): exact tagged 2D boundary loops
(`loops2d`, via `surfaces::boundary_outline`) and the analytic sizing field
(`sizing`). It obeys the interior-exactness contract: no positive field
value is ever consumed as a distance.

### 4.5 `app` — the application

eframe + egui_dock shell (theme ported from `app/assets/theme.qss`; the
toolbar shows the casoCAD wordmark from the vector
`app/assets/casocad-wordmark.svg`, rasterized by the `egui_extras` SVG
loader at the on-screen pixel size so it stays crisp at any DPI): Scene
and Properties panels tabified left, Log panel, central viewport shown as an
offscreen texture. `state.rs` (selection, 50-snapshot undo/redo, LengthUnit
registry), `dimensions.rs` (full measurement-entry parser: units incl.
in/ft/"/', arithmetic expressions, multi-value entry), `tools.rs`
(Select / drag-create / point-create / Move / Rotate on the grid; all 1D
kinds — segment, polyline, bezier curve, tubes — are point tools; while
drawing, a live geometry ghost is built by the *same* kernel builder that
commits (`add_primitive_from_drag` / `add_point_shape_from_world_points`
on a throwaway `SceneDocument`), tessellated through the regular surface
router and shown as an overlay — dimension category styles it only
(3D translucent fill via per-surface `ViewportSurface::alpha` + the
renderer's blended-chunk pass, 2D outline, 1D wires), so preview can never
drift from the committed result and new kinds need zero preview code),
`boundary_tool.rs` (BoundaryRegion hover tool + BoundaryCutter with three
knife kinds — segment, polygon, bezier surface — and split preview; region
highlights filter whole display
triangles by classifier criteria 1-3, then clip the survivors exactly
against each cut volume with root-found seam vertices via
`surfaces::clipping` — the highlight seam lies on the knife's true zero set
regardless of display tessellation, the cyan/orange split previews share
a bitwise-identical seam, and every area knife is bounded to the clicked
sheet so closed surfaces never show an antipodal phantom cut; see
`design_docs/boundary_cutter_exactness.md`),
`meshing_panel.rs` + `script_runner.rs`
(Rhai scripting: `domains`, per-domain SDF/region queries, `mesh` MeshIR
builder, 2D `mesh_space()` helpers; full script reference in
`docs/mesher_script_api.md`; preview overlay; `.arrow` export and
MeshIR converter exports such as SU2 — browser export is a Blob download).

**Unified tool-interaction grammar.** Every viewport tool follows one
lifecycle, enforced by a single dispatcher instead of per-tool key handling:

- **Commit model is per input class**: drag-created kinds commit at gesture
  end (typed dimensions are applied at mouse release); point-placed kinds
  and the cutter commit on **Enter**; Measure commits on its second click;
  Boundary Region commits on click. There is no draft/pending object state —
  undo is the safety net.
- **Tools stay armed** after a commit for repeated use; Esc (idle) or
  re-clicking the active tool button/menu entry exits to Select.
- **The key ladder**: **Esc** first clears pending input (typed dimensions,
  placed points, the pending measure point, a live Move/Rotate gesture —
  reverted via `AppState::abort_to_last_snapshot`, which leaves no redo
  entry), and a second Esc exits to Select. **Enter** confirms whatever is
  pending. **Backspace** pops the last placed point / typed character.
- **One dispatcher**: `ToolState::{has_pending, clear_pending,
  confirm_pending, pop_pending}` called once from the app `shortcuts()`
  pass; tool handlers never read Esc/Enter/Backspace themselves. The
  Measure pending point lives in `ToolState::points` like every point tool
  (only committed annotations stay in the viewport panel).
- **Documented exceptions**: Delete while measuring is annotation-scoped
  (clears measurements, never scene objects); document edit shortcuts
  (undo/redo/copy/paste/duplicate/delete) are suppressed only while input
  is pending — an armed-but-idle tool no longer blocks them; a focused text
  field owns all keys.

Viewport navigation overlays (`viewport_panel.rs`, ported from the Python
`_OrientationWidget` / `_build_view_panel`): a bottom-left axis triad
(world X/Y/Z projected with the camera basis, back-to-front with
depth-dimmed alpha) and bottom-center reference-view buttons
`3D / {x,y} / {x,z} / {y,z}` (keys 1–4). Each button sets the rendered
grid plane (`RenderOptions::grid_plane`) and flies the camera to the view
with a 260 ms smoothstep; manual orbit cancels the flight. Unlike the
Python app, the drawing tools still work on the `Z = 0` XY plane — the
plane switch is visualization-only for now.

### 4.6 `web`

`index.html` hosting the wasm-pack output (`pkg/`), canvas id
`casowasm_canvas`. WebGPU with WebGL2 fallback via wgpu.

## 5. Parity Harness And Verification

Golden files are generated by the Python kernel (read-only use of the
casoCAD `.venv`) and replayed by Rust tests:

- `tools/export_goldens.py` → `kernel/tests/parity.rs`: **52 fixtures /
  ~11.5k sampled SDF values, ≤1e-12 relative tolerance** against
  `to_numpy()`.
- `kernel/tests/roundtrip.rs`: the default-scene resave matches its golden
  (JSON Value equality); save/load/save is a fixed point.
- `tools/export_surface_goldens.py` → `surfaces/tests/surface_parity.rs`:
  **44 golden metric rows** (status / vertex / triangle / wire counts + max
  surface error, at res 12 and 96) match Python exactly.
- Boundary/meshing: ports of `test_boundary_region_classifier.py`,
  `test_boundary_split.py`, `test_mesh_api.py`; MeshIR `.arrow` artifacts
  round-trip through the Rust Arrow reader/writer with shared topology intact.

The goldens are frozen: regenerating them requires the Python exporters
(`tools/export_goldens.py` at the `python-final` tag). When adding a node
kind, extend `build_fixture` in `kernel/tests/parity.rs` and add the
serialization round-trip case; new-kind goldens need the archived exporter.

Standing checks (all green as of this document):

```bash
cargo test                                   # 54 tests
cargo clippy --all-targets                   # clean
cargo build --target wasm32-unknown-unknown  # wasm32 must always build
```

Build and run:

```bash
cargo run -p caso-app                                  # native desktop
cd app && wasm-pack build --target web --out-dir ../web/pkg
python3 -m http.server 8642 --directory ../web         # then open in browser
```

Headless browser smoke: `google-chrome --headless=new
--use-angle=swiftshader-webgl --virtual-time-budget=25000 --screenshot=...`.

## 6. Current Status

All seven port phases are **done**; every gate passed, including Python
interop (scene.json and `.arrow` round-trips).

1. **Kernel** — all primitives/operators/transforms/generators, roles,
   preconditions, model compile. Gate: 52-fixture / 11.5k-sample parity.
2. **Scene document + serialization** — id-based DAG, all edit ops, scene.json
   v1 with legacy migration. Gate: byte-stable round-trips through Python.
3. **Surface builders** — analytic tessellation, exact clipping, dual
   contouring, 2D fills, cache. Gate: 44 golden metric rows exact.
4. **wgpu viewport** — native and browser (WebGPU boots panic-free; WebGL2
   verified by headless screenshot: von Kármán scene + grid + axes + UI).
5. **Editing UI** — panels, theme, toolbar/menus, drag- and point-create
   tools with typed dimensions, Move/Rotate, extrude/revolve, booleans,
   copy/paste/duplicate, undo/redo, unit switch, save/load (web save =
   download), Validate, Log. Gate: the acceptance workflow (draw box, draw
   cylinder, subtract, set fluid domain, save) opens identically in Python
   casoCAD.
6. **Boundary regions + cutter** — classifier, patches, picking, hover tool,
   four knife kinds with split preview, region properties. Gate: classifier
   and split test ports green.
7. **Meshing workspace** — `MeshableDomain` API, Rhai scripting page, per-tag
   preview overlay, Arrow IPC export readable by Python.

## 7. Future View

### 7.1 Aspect polish (remaining, not gate items)

- Viewport chrome: orientation gizmo, view panel (3D/xy/xz/yz), monospace
  readout.
- Boolean menus with live hover previews.
- WASD fly navigation; X-ray / components toggle; background color picker;
  snap control.
- Scene load in the browser (file upload; save/download already works).
- Primitive SVG icons in the scene tree.

### 7.2 Product direction

- Solver case generation — OpenLB and Palabos first (regular lattice), then
  OpenFOAM/SU2 — does not exist in the Python app yet either; when it lands,
  it belongs behind the `meshing`/`kernel` API here.
- A 2D project remains authoritatively 2D; solver adapters may generate thin
  topology, the user never extrudes CAD geometry for a solver.

### 7.3 Explicitly out of scope

- Resurrecting the deleted GPU raymarcher / per-scene shader codegen: the
  project settled on exact display surfaces; the surface pipeline is what
  ports.

## 8. Porting Gotchas (keep for future work)

- **egui 0.35 API**: `App::ui(&mut self, ui, frame)` (not `update(ctx)`);
  `egui::Panel::top(id).show(ui, ..)`; `ctx.all_styles_mut`;
  `input.smooth_scroll_delta()`; panels use `.default_size` (not
  `.default_width`); context menus close via `ui.close()`.
- **wgpu 29 API**: `bind_group_layouts: &[Some(&layout)]`,
  `immediate_size: 0`, `depth_write_enabled: Some(bool)`,
  `multiview_mask: None` also on `RenderPassDescriptor`. Avoid
  `mappedAtCreation` on the web — use `queue.write_buffer` (see
  `upload_buffer()`).
- **WGSL**: `fwidth` requires uniform control flow — restructure branches
  around it.
- **wasm32**: `std::time::Instant` panics — cfg-gate any timing.
  `getrandom` (pulled in by rhai) needs `.cargo/config.toml` rustflags
  `getrandom_backend="wasm_js"` plus the `wasm_js` feature.
- **Rhai**: fallible registered functions must return
  `Result<T, Box<EvalAltResult>>` — `Result<T, String>` registers as a plain
  value.
- **Scene load renumbers ids deterministically** (the fluid root follows its
  object); never persist ObjectIds across a load.
- The viewport is an offscreen texture registered through
  `render_state.renderer.write().register_native_texture` and drawn with
  `ui.painter().image`.
